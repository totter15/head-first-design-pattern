# 커맨드 패턴

## 개념

커맨드 패턴은 요청 내역을 객체로 캡슐화해서, 객체를 서로 다른 요청 내역에 따라 매개변수화할 수 있게 해주는 패턴이다.  
핵심은 "요청하는 쪽(인보커)"과 "실제 작업을 수행하는 쪽(리시버)"을 분리하는 것이다.

요청을 객체로 감싸면 큐에 저장하거나, 로그를 기록하거나, 작업 취소(undo) 기능을 구현할 수 있다.

## 구조

**Client (클라이언트)**

ConcreteCommand 객체를 생성하고 Receiver를 설정한다.

**Invoker (인보커)**

명령이 들어 있으며, `execute()` 메서드를 호출함으로써 커맨드 객체에게 특정 작업 수행을 요구한다. (`SimpleRemoteControl`, `RemoteControl`)

**Command (커맨드 인터페이스)**

모든 커맨드 객체가 구현해야 하는 공통 인터페이스 (`execute()`, `undo()`)

**ConcreteCommand (구상 커맨드)**

특정 행동과 리시버를 연결한다. 인보커에서 `execute()`를 호출하면 리시버의 메서드를 호출해서 작업을 처리한다. (`LightonCommand`, `LightOffCommand`, `StereoOnWithCDCommand` 등)

**Receiver (리시버)**

요구 사항을 수행할 때 어떤 일을 처리해야 하는지 알고 있는 객체 (`Light`, `Stereo`)

```txt
Client
  └─ ConcreteCommand 생성 + Receiver 설정

SimpleRemoteControl (Invoker)
  ├─ setCommand()       // 커맨드 객체 저장
  └─ buttonWasPressed() // execute() 호출

Command (Interface)
  ├─ execute(): void
  └─ undo(): void

LightonCommand / LightOffCommand / StereoOnWithCDCommand (ConcreteCommand)
  └─ execute() 구현 → Receiver 메서드 호출

Light / Stereo (Receiver)
  └─ 실제 작업 수행 (on, off, setCD, setVolume ...)
```

> "무엇을 실행할지"는 ConcreteCommand가 캡슐화하고, "언제 실행할지"는 Invoker가 결정한다.

## 예시 1 — 기본 커맨드 객체 만들기

리모컨(인보커)으로 조명(리시버)을 제어하는 예시

1. Command 인터페이스

```ts
interface Command {
  execute(): void;
  undo(): void;
}
```

2. Receiver

```ts
class Light {
  name: string;

  constructor(name: string) {
    this.name = name;
  }

  on(): void {
    console.log(`${this.name} light on`);
  }

  off(): void {
    console.log(`${this.name} light off`);
  }
}
```

3. ConcreteCommand

```ts
class LightonCommand implements Command {
  light: Light;

  constructor(light: Light) {
    this.light = light;
  }

  execute(): void {
    this.light.on();
  }

  undo(): void {
    this.light.off();
  }
}
```

4. Invoker

```ts
class SimpleRemoteControl {
  slot: Command;

  setCommand(command: Command) {
    this.slot = command;
  }

  buttonWasPressed() {
    this.slot.execute();
  }
}
```

5. 사용

```ts
const remote = new SimpleRemoteControl(); // 인보커
const light = new Light("Living Room"); // 리시버
const lightOn = new LightonCommand(light); // 커맨드 객체 (행동 + 리시버)

remote.setCommand(lightOn); // 커맨드 객체를 인보커에게 전달
remote.buttonWasPressed(); // execute() → light.on()
```

## 예시 2 — 슬롯에 명령 할당하기

여러 기기를 제어하는 리모컨 확장. 7개의 On/Off 슬롯을 가진 리모컨을 구현한다.

1. NoCommand (Null Object 패턴)

빈 슬롯에 기본값으로 할당해서 null 체크를 없앤다.

```ts
class NoCommand implements Command {
  execute(): void {
    console.log("No command");
  }
  undo(): void {
    console.log("No command");
  }
}
```

2. 확장된 Invoker

```ts
class RemoteControl {
  onCommands: Command[];
  offCommands: Command[];

  constructor() {
    this.onCommands = new Array<Command>(7);
    this.offCommands = new Array<Command>(7);

    for (let i = 0; i < 7; i++) {
      this.onCommands[i] = new NoCommand();
      this.offCommands[i] = new NoCommand();
    }
  }

  setCommand(slot: number, onCommand: Command, offCommand: Command) {
    this.onCommands[slot] = onCommand;
    this.offCommands[slot] = offCommand;
  }

  onButtonWasPushed(slot: number) {
    this.onCommands[slot]?.execute();
  }

  offButtonWasPushed(slot: number) {
    this.offCommands[slot]?.execute();
  }
}
```

3. 추가 Receiver + ConcreteCommand

```ts
class Stereo {
  on(): void {
    console.log("Stereo on");
  }
  off(): void {
    console.log("Stereo off");
  }
  setCD(): void {
    console.log("Stereo set CD");
  }
  setVolume(volume: number): void {
    console.log(`Stereo set volume to ${volume}`);
  }
}

class StereoOnWithCDCommand implements Command {
  stereo: Stereo;

  constructor(stereo: Stereo) {
    this.stereo = stereo;
  }

  execute(): void {
    this.stereo.on();
    this.stereo.setCD();
    this.stereo.setVolume(11);
  }

  undo(): void {
    this.stereo.off();
  }
}
```

4. 사용

```ts
const remoteControl = new RemoteControl();

const livingRoomLight = new Light("Living Room");
const kitchenLight = new Light("Kitchen");
const stereo = new Stereo();

remoteControl.setCommand(
  0,
  new LightonCommand(livingRoomLight),
  new LightOffCommand(livingRoomLight),
);
remoteControl.setCommand(
  1,
  new LightonCommand(kitchenLight),
  new LightOffCommand(kitchenLight),
);
remoteControl.setCommand(
  2,
  new StereoOnWithCDCommand(stereo),
  new StereoOffWithCDCommand(stereo),
);

remoteControl.onButtonWasPushed(0); // Living Room light on
remoteControl.offButtonWasPushed(0); // Living Room light off
remoteControl.onButtonWasPushed(2); // Stereo on → set CD → volume 11
```

## 예시 3 — 작업 취소(Undo) 기능

`Command` 인터페이스에 `undo()` 메서드를 추가하고, 마지막으로 실행된 커맨드를 기억해서 되돌린다.

```ts
class RemoteControlWithUndo {
  onCommands: Command[];
  offCommands: Command[];
  undoCommand: Command;

  constructor() {
    this.onCommands = new Array<Command>(7);
    this.offCommands = new Array<Command>(7);
    this.undoCommand = new NoCommand();
  }

  setCommand(slot: number, onCommand: Command, offCommand: Command) {
    this.onCommands[slot] = onCommand;
    this.offCommands[slot] = offCommand;
  }

  onButtonWasPushed(slot: number) {
    this.onCommands[slot]?.execute();
    this.undoCommand = this.onCommands[slot] ?? new NoCommand();
  }

  offButtonWasPushed(slot: number) {
    this.offCommands[slot]?.execute();
    this.undoCommand = this.offCommands[slot] ?? new NoCommand();
  }

  undoButtonWasPushed() {
    this.undoCommand?.undo();
  }
}
```

`onButtonWasPushed()` / `offButtonWasPushed()` 실행 시 마지막 커맨드를 `undoCommand`에 저장하고,  
`undoButtonWasPushed()`를 호출하면 해당 커맨드의 `undo()`가 실행되어 이전 상태로 되돌린다.

## 언제 사용하면 좋은가?

**✔ 요청을 객체로 캡슐화해야 할 때**

실행할 작업을 매개변수화하거나, 큐에 넣어 순서대로 실행하고 싶을 때

**✔ 실행 취소(Undo) / 재실행(Redo) 기능이 필요할 때**

각 커맨드에 `undo()`를 구현하면 히스토리 기반 되돌리기가 가능하다

**✔ 요청하는 쪽과 수행하는 쪽을 분리하고 싶을 때**

인보커는 리시버의 구체적인 동작을 몰라도 `execute()`만 호출하면 된다

**✔ 매크로 커맨드(여러 명령 묶기)가 필요할 때**

여러 커맨드를 하나의 커맨드로 묶어서 한 번에 실행할 수 있다
